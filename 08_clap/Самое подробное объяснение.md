## 1️⃣ Импорты

```rust
// clap::Parser — это трейт (набор методов), который добавляет структуре
// возможность парсить аргументы командной строки.
// Без него нельзя вызвать .parse() на нашей структуре Cli.
use clap::Parser;

// clap::ValueEnum — трейт для перечислений (enum).
// Он говорит clap: "значение этого enum должно быть одним из перечисленных вариантов".
// clap сам проверит, что пользователь ввёл допустимое значение,
// и выдаст ошибку с подсказкой (например, "possible values: ru, en").
use clap::ValueEnum;
```

### 💡 Почему именно так?

`clap` — это "двигатель" для CLI. Он берёт на себя всю рутину:
- Читает и проверяет то, что ввёл пользователь в терминале
- Генерирует красивый `--help`
- Проверяет типы аргументов

Без `Parser` нельзя вызвать `.parse()`. Без `ValueEnum` нельзя сделать строгий список вариантов.

## 2️⃣ Перечисление `Language`

```rust
// #[derive(ValueEnum, Clone, Debug)] — атрибуты, которые заставляют компилятор
// автоматически реализовать для enum Language указанные трейты:
// - ValueEnum: позволяет clap проверять допустимые варианты
// - Clone: позволяет копировать значение (нужно для default_value_t)
// - Debug: позволяет печатать через {:?} (полезно для verbose-режима)
#[derive(ValueEnum, Clone, Debug)]
enum Language {
    Ru,   // вариант "ru" (команда --language ru)
    En,   // вариант "en"
    // De, // можно добавить, но пока закомментировано
    // It, // можно добавить
}
```

### 💡 Почему именно так?

Использование `enum` + `ValueEnum` даёт **два преимущества**:
1. ✅ **Компилятор проверяет** — опечатка в `mainnet` не пройдёт
2. ✅ **Автогенерация `--help`** — clap сам покажет список вариантов

Альтернатива (плохая): принимать `String` и проверять в `main` — больше кода, ошибки позже.

## 3️⃣ Структура `Cli` — описание аргументов

```rust
// #[derive(Parser)] — это главная "магия".
// Компилятор генерирует код, который умеет читать аргументы командной строки
// и заполнять поля структуры.
#[derive(Parser)]
// #[command(...)] — даёт метаинформацию о программе.
// Эти данные попадают в --help.
#[command(
    name = "greet",           // имя программы (показывается в строке Usage)
    version = "1.0",          // версия (выводится по флагу --version)
    about = "Приветствие с валидацией" // краткое описание
)]
struct Cli {
    // ══════════════════════════════════════════════
    // ПОЛЕ 1: name — позиционный аргумент
    // ══════════════════════════════════════════════
    /// Имя (позиционный аргумент)
    // Поле name типа String. Позиционный аргумент означает,
    // что пользователь просто пишет значение после имени программы, без флага.
    // Например: ./greet Alice
    // Это обязательный аргумент.
    name: String,

    // ══════════════════════════════════════════════
    // ПОЛЕ 2: language — опция с дефолтом
    // ══════════════════════════════════════════════
    /// Язык (флаг -l или --language)
    // Атрибут #[arg(...)] настраивает парсинг для этого поля.
    // - short, long: разрешены флаги -l и --language.
    // - value_enum: значение должно быть из enum Language.
    // - default_value_t = Language::Ru: если язык не указан,
    //   подставляется Language::Ru.
    #[arg(short, long, value_enum, default_value_t = Language::Ru)]
    language: Language,

    // ══════════════════════════════════════════════
    // ПОЛЕ 3: city — обязательная опция
    // ══════════════════════════════════════════════
    /// Город (обязательная опция -c или --city)
    // Здесь нет default_value_t, и тип не Option => clap считает этот аргумент
    // обязательным. Пользователь обязан написать --city Москва или -c Москва.
    #[arg(short, long)]
    city: String,

    // ══════════════════════════════════════════════
    // ПОЛЕ 4: age — необязательная опция
    // ══════════════════════════════════════════════
    /// Возраст (опция -a или --age, только 0..120)
    // Тип Option<u8> означает, что аргумент необязательный.
    // value_parser = clap::value_parser!(u8).range(0..=120) — валидатор:
    //   - сначала значение парсится как целое число u8,
    //   - затем проверяется, что оно входит в диапазон 0..120.
    #[arg(short, long, value_parser = clap::value_parser!(u8).range(0..=120))]
    age: Option<u8>,

    // ══════════════════════════════════════════════
    // ПОЛЕ 5: verbose — булев флаг
    // ══════════════════════════════════════════════
    /// Подробный вывод (флаг -v или --verbose)
    // Поле типа bool. Если флаг -v или --verbose присутствует, verbose = true.
    // Если флаг отсутствует, verbose = false (по умолчанию).
    #[arg(short, long)]
    verbose: bool,
}
```

### 💡 Почему каждое поле оформлено именно так?

| Поле | Тип | Решение | Почему |
|------|-----|---------|-------|
| `name` | `String` | **Обязательный** | Без имени приветствие не имеет смысла |
| `language` | `Language` | **Дефолт `Ru`** | Удобно: не нужно писать каждый раз `--language ru` |
| `city` | `String` | **Обязательный** | Город критичен для вывода, нет разумного дефолта |
| `age` | `Option<u8>` | **Необязательный** | Возраст не всегда нужен, нет разумного дефолта (0 или 18 — ложь) |
| `verbose` | `bool` | **Флаг** | Переключатель "вкл/выкл", естественный дефолт = false |

## 4️⃣ Функция `main`

```rust
fn main() {
    // ══════════════════════════════════════════════
    // ВОЛШЕБНАЯ СТРОКА: Cli::parse()
    // ══════════════════════════════════════════════
    // Cli::parse() делает следующее:
    // 1) Читает аргументы из std::env::args()
    // 2) Разбирает их согласно описанию структуры
    // 3) Если ошибка — печатает сообщение и завершает программу
    // 4) Если всё хорошо — возвращает Cli с заполненными полями
    let args = Cli::parse();

    // ══════════════════════════════════════════════
    // Выбираем приветствие по языку
    // ══════════════════════════════════════════════
    // args.language — это значение enum Language (Ru, En).
    // match сравнивает его с вариантами и возвращает соответствующий &str.
    let greeting = match args.language {
        Language::Ru => "Привет",
        Language::En => "Hello",
    };

    // ══════════════════════════════════════════════
    // Подробный режим (verbose)
    // ══════════════════════════════════════════════
    // args.verbose — это bool: true если пользователь указал -v или --verbose.
    if args.verbose {
        println!("[VERBOSE] Язык: {:?}", args.language);
        println!("[VERBOSE] Имя: {}", args.name);
        println!("[VERBOSE] Город: {}", args.city);
        
        // if let Some(age) = args.age — безопасное извлечение значения из Option.
        // Если age = Some(число), то переменная age получает это число,
        // и выполняется блок кода. Если None — блок пропускается.
        if let Some(age) = args.age {
            println!("[VERBOSE] Возраст: {}", age);
        }
    }

    // ══════════════════════════════════════════════
    // Основной вывод
    // ══════════════════════════════════════════════
    println!("{}, {}! Твой город: {}", greeting, args.name, args.city);
    
    // Если возраст был указан — выводим дополнительную фразу
    if let Some(age) = args.age {
        println!("Тебе {} лет.", age);
    }
}
```

## 5️⃣ Категория «Что если?» — как принимать решение

Разработчик CLI должен ответить на **три вопроса** для каждого аргумента:

### ❓ Вопрос 1: Должен ли пользователь ОБЯЗАТЕЛЬНО указать этот аргумент?

| Ответ | Как оформить |
|-------|-------------|
| **ДА** | Тип `T` (не `Option<T>`), НЕ ставить `default_value` |
| **НЕТ** | → переходим к вопросу 2 |

### ❓ Вопрос 2: Есть ли РАЗУМНОЕ значение по умолчанию?

| Ответ | Как оформить |
|-------|-------------|
| **ДА** | Ставим `default_value_t`, тип `T` (не `Option<T>`) |
| **НЕТ** | Используем `Option<T>` без `default_value` |

### ❓ Вопрос 3: Это просто ФЛАГ (вкл/выкл)?

| Ответ | Как оформить |
|-------|-------------|
| **ДА** | Тип `bool`, атрибут `#[arg(short, long)]` |

## 6️⃣ Таблица принятия решений

> Вопрос: **"Что если пользователь не указал аргумент?"**

| Тип поля | `default_value` | Результат |
|----------|----------------|-----------|
| `T` | ❌ нет | 🛑 **Ошибка**, программа не запустится |
| `T` | ✅ есть | ✅ Будет использовано значение по умолчанию |
| `Option<T>` | ❌ нет | ✅ `args.field == None` |
| `Option<T>` | ✅ есть | Редко, но можно: дефолт применяется, но тип остаётся `Option` |
| `bool` | — | ✅ `false`, если флаг отсутствует |

### 🎯 Практический совет

| Ситуация | Решение |
|----------|---------|
| Аргумент критичен для логики | **Обязательный** (без `Option`) |
| Аргумент можно разумно угадать | **`default_value_t`** |
| Аргумент нужен не всегда, нет разумного дефолта | **`Option<T>`** |
| Аргумент — просто переключатель | **`bool`** |

## 7️⃣ Примеры запуска и ошибки

### ✅ Пример 1: Правильный запуск со всеми опциями

```bash
$ cargo run -- Alice --city Moscow --language en --age 25 --verbose
```

**Вывод:**
```
[VERBOSE] Язык: En
[VERBOSE] Имя: Alice
[VERBOSE] Город: Moscow
[VERBOSE] Возраст: 25
Hello, Alice! Твой город: Moscow
Тебе 25 лет.
```

---

### ✅ Пример 2: Запуск без возраста и без verbose

```bash
$ cargo run -- Bob --city SPb
```

**Вывод:**
```
Привет, Bob! Твой город: SPb
```

> Возраст не выводится, verbose молчит. `language` = Ru (дефолт).

---

### ❌ Пример 3: Ошибка — забыли `--city`

```bash
$ cargo run -- Charlie
```

**Ошибка от clap (до main):**
```
error: the following required arguments were not provided: --city <CITY>
Usage: greet <NAME> --city <CITY> ...
```

---

### ❌ Пример 4: Ошибка — неверное значение языка

```bash
$ cargo run -- Dave --city London --language fr
```

**Ошибка от clap:**
```
error: 'fr' is not a valid value for '--language <LANGUAGE>'
Possible values: ru, en
```

---

### ❌ Пример 5: Ошибка — возраст вне диапазона

```bash
$ cargo run -- Eve --city NY --age 150
```

**Ошибка от clap:**
```
error: value validation failed for '--age <AGE>': 150 is not in 0..=120
```

---

### ❌ Пример 6: Ошибка — неизвестный флаг

```bash
$ cargo run -- Frank --city Berlin --verbose --unknown
```

**Ошибка от clap:**
```
error: unexpected argument '--unknown' found
```

## 📖 Резюме

| Концепт | Суть |
|---------|------|
| **`#[derive(Parser)]`** | Магия: структура → парсер CLI |
| **Позиционный аргумент** | Просто значение после команды (без флага) |
| **Опция (short/long)** | Аргумент с флагом `-n` или `--name` |
| **Флаг (bool)** | Переключатель без значения (просто присутствует/отсутствует) |
| **`default_value_t`** | Значение по умолчанию, если пользователь пропустил аргумент |
| **`Option<T>`** | Аргумент может быть, а может не быть |
| **`ValueEnum`** | Строгий список допустимых значений |
| **`value_parser`** | Проверка значения на лету (диапазон, формат) |
| **`.parse()`** | Запускает парсинг и проверку |

---

> 🚀 **Главный совет:** Начинай с `#[derive(Parser)]` и простых типов (`String`, `bool`, `u32`).
> Добавляй `ValueEnum` и `value_parser` когда понадобится проверка ввода.
> Не пытайся выучить всё сразу — пользуйся, и навык придёт сам!

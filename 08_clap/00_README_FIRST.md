# 📖 READ ME FIRST — Что такое CLI и зачем нужен clap?

## 🤔 Ты когда-нибудь запускал программу в терминале?

```bash
ls -la /home
```
Или:
```bash
cargo build --release
```

Вот эти `-la` и `--release` — это и есть **аргументы командной строки** (CLI args).

## 🧠 Простыми словами

Представь, что ты заказываешь пиццу:
- Ты говоришь: **«Мне одну большую пепперони, пожалуйста»**
- `одну` = количество
- `большую` = размер
- `пепперони` = тип

Это и есть **аргументы**. Программа (пиццерия) их получает и знает, что тебе нужно.

## 💻 Как это выглядит в коде без clap

Когда ты запускаешь программу:
```bash
./myapp Артём
```

Rust получает это как список строк:
```rust
let args = std::env::args(); // ["myapp", "Артём"]
```

И тебе нужно **самому** разбирать этот список. Проверять, что `Артём` — это строка, что она не пустая, что нет лишних аргументов... Это **нудно** и **ошибочно**.

## ✨ Что делает clap

**clap** — это библиотека, которая делает всю эту работу **за тебя**.

Ты просто говоришь:
```rust
struct Args {
    name: String,
    #[arg(short, long)]
    verbose: bool,
}
```

А clap сам:
1. ✅ Разбирает `std::env::args()`
2. ✅ Проверяет типы (строка, число, bool)
3. ✅ Генерирует `--help`
4. ✅ Сообщает об ошибках

## 🎯 Что ты научишься

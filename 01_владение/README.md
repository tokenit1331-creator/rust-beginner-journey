# 🔥 ПОЛНЫЙ УРОК: Владение (Ownership)

## 🧠 Суть (на пальцах)

Представь, что у тебя есть **ключ от квартиры**. Ключ — это твои данные в памяти.

**Правило №1:** У каждого ключа может быть **только один хозяин**.
Если ты передал ключ другу — у тебя ключа больше нет. Ты не можешь зайти в квартиру.

**Правило №2:** Ты можешь **одолжить** ключ другу на время.
Пока ты дал ключ — ты сам не можешь им пользоваться. Но когда друг вернёт ключ — ты снова хозяин.

**Правило №3:** Много людей могут **смотреть** на ключ одновременно.
Но только один человек может **менять замок** (и только если у него уникальный доступ).

Это ВСЁ владение. Три правила. Всё остальное — детали.

---

## 📋 Три правила владения

| Правило | Что означает | Аналогия |
|---------|-------------|----------|
| **1. У каждого значения один владелец** | Одна переменная — один «хозяин» данных | Один ключ от квартиры |
| **2. Владение можно передать (move)** | При присваивании или передаче в функцию владение переходит | Отдал ключ другу — у тебя больше нет |
| **3. Владение заканчивается** | Когда владелец выходит из области видимости — данные удаляются | Квартиру снесли, когда ты съехал |

# 📘 ПОДРОБНЫЙ РАЗБОР

## 1️⃣ Правило №1: Один владелец

```rust
let s1 = String::from("Привет"); // s1 — владелец строки
let s2 = s1;                      // Владение ПЕРЕШЛО к s2
                                  // s1 БОЛЬШЕ НЕ ВЛАДЕЕТ строкой!

// println!("{}", s1); // ❌ ОШИБКА! s1 больше не владелец
println!("{}", s2);    // ✅ Работает, s2 — новый владелец
```

### ❓ Почему так?

Rust удаляет данные из памяти, когда владелец выходит из области видимости.
Если бы у `s1` и `s2` было право удалять одну и ту же строку — это **double-free** (дважды освободить память).
Это одна из главных уязвимостей в C и C++. Rust решает её на уровне компилятора.

---

## 2️⃣ Правило №2: Move (передача владения)

### Пример 1: Передача в функцию

```rust
fn main() {
    let name = String::from("Alice");
    greet(name);
    // println!("{}", name); // ❌ name больше не владеет строкой!
    //    move произошёл при передаче в greet()
}

fn greet(name: String) {
    println!("Привет, {}!", name);
    // Здесь name выходит из области видимости
    // Память освобождается ✅
}
```

### Пример 2: Возврат владения

```rust
fn main() {
    let name = String::from("Alice");
    let name = greet(name); // возвращаем владение обратно
    println!("{}", name);  // ✅ Работает! name снова владеет
}

fn greet(name: String) -> String {
    println!("Привет, {}!", name);
    name // возвращаем владение вызывающему
}
```

### 💡 Совет опытного разработчика №1

> Возвращать владение через `-> String` — муторно. Для этого есть **заимствование** (см. ниже).
> Но понимать механику move — обязательно. Это фундамент.

## 3️⃣ Copy и Clone — исключения

### Типы Copy (копируются, а не перемещаются)

Некоторые типы настолько простые, что Rust может их **скопировать**, а не перемещать:

```rust
let x = 42;     // i32 — тип Copy
let y = x;      // КОПИРУЕТСЯ, а не перемещается
println!("{}", x); // ✅ Всё ещё работает!
println!("{}", y); // ✅ И это работает
```

Типы Copy: `i32`, `f64`, `bool`, `char`, кортежи из Copy-типов

### Clone — явное копирование

Для типов, которые НЕ Copy (например, `String`), можно сделать явную копию:

```rust
let s1 = String::from("Привет");
let s2 = s1.clone(); // Явная копия (глубокое копирование)

println!("{}", s1); // ✅ Работает! s1 всё ещё владеет
println!("{}", s2); // ✅ И s2 владеет своей копией
```

### ⚠️ Когда НЕ использовать clone

```rust
// ❌ ПЛОХО: clone без необходимости
fn process(data: Vec<String>) {
    for item in data.clone().iter() { // Зачем клонировать?
        println!("{}", item);
    }
}

// ✅ ХОРОШО: просто взять ссылку
fn process(data: &[String]) {
    for item in data.iter() {
        println!("{}", item);
    }
}
```

### 💡 Совет опытного разработчика №2

> `clone()` — это костыль. Если ты постоянно пишешь `.clone()`, значит ты не понимаешь заимствование.
> Изучи `&` — это спасёт тебя и твою память.

## 4️⃣ Заимствование (Borrowing) — ССЫЛКИ

### Что такое ссылка?

Ссылка (`&T`) — это «взглянуть на данные, не забирая ключ». Владелец остаётся прежним, мы просто **одалживаем** доступ.

### &T — неизменяемая ссылка (чтение)

```rust
fn main() {
    let s1 = String::from("Привет");
    let len = calculate_length(&s1); // передаём ССЫЛКУ, а не строку
    println!("'{}' имеет длину {}.", s1, len); // ✅ s1 всё ещё владеет!
}

fn calculate_length(s: &String) -> usize {
    s.len() // s — ссылка, но можно читать
} // s выходит из области, но так как это ссылка — память НЕ освобождается
```

### &mut T — изменяемая ссылка (запись)

```rust
fn main() {
    let mut s = String::from("Привет");
    add_world(&mut s);
    println!("{}", s); // "Привет, мир!"
}

fn add_world(s: &mut String) {
    s.push_str(", мир!"); // изменяем через ссылку
}
```

### 🚫 Правила заимствования (ЗАПОМНИ!)

| Ситуация | Разрешено? |
|----------|-----------|
| Одна изменяемая ссылка (`&mut T`) | ✅ Да |
| Много неизменяемых ссылок (`&T`) | ✅ Да |
| Одна `&mut T` + одна `&T` **одновременно** | ❌ **НЕТ!** |
| Две `&mut T` **одновременно** | ❌ **НЕТ!** |

```rust
let mut s = String::from("Привет");

let r1 = &s;      // ✅ одна неизменяемая
let r2 = &s;      // ✅ вторая неизменяемая — норм
// let r3 = &mut s; // ❌ ОШИБКА! нельзя &mut при &T
println!("{}, {}", r1, r2); // r1 и r2 больше не используются

let r3 = &mut s;  // ✅ теперь можно — старых ссылок нет
println!("{}", r3);
```

### 💡 Совет опытного разработчика №3

> Ошибка «cannot borrow as mutable because it is also borrowed as immutable» — САМАЯ ЧАСТАЯ у новичков.
> Решение: **сузь область видимости ссылок**. Сделай так, чтобы `&T` перестали использоваться ДО того, как появится `&mut T`.

## 5️⃣ Реальные примеры из крипто-кода 🪙

### Пример 1: перевод токенов (Move)

```rust
struct Wallet {
    address: String,
    balance: f64,
}

fn main() {
    let alice = Wallet {
        address: String::from("0xAlice"),
        balance: 100.0,
    };
    
    // ❌ ПЛОХО: передача владения без нужды
    send_all_tokens(alice); // alice больше не существует!
    // println!("{}", alice.address); // ОШИБКА!
}

fn send_all_tokens(wallet: Wallet) {
    println!("Отправляем {} из {}", wallet.balance, wallet.address);
}
```

```rust
// ✅ ХОРОШО: используем ссылку
fn main() {
    let alice = Wallet {
        address: String::from("0xAlice"),
        balance: 100.0,
    };
    
    check_balance(&alice);        // & — только читаем
    println!("Баланс: {}", alice.balance); // ✅ alice всё ещё жива
}

fn check_balance(wallet: &Wallet) {
    println!("Баланс кошелька {}: {} USDT", wallet.address, wallet.balance);
}
```

### Пример 2: мутация баланса (&mut T)

```rust
struct Wallet {
    address: String,
    balance: f64,
}

fn main() {
    let mut alice = Wallet {
        address: String::from("0xAlice"),
        balance: 100.0,
    };
    
    let success = transfer(&mut alice, 50.0, "0xBob");
    
    if success {
        println!("Новый баланс: {} USDT", alice.balance); // ✅ 50.0
    }
}

fn transfer(wallet: &mut Wallet, amount: f64, to: &str) -> bool {
    if wallet.balance >= amount {
        wallet.balance -= amount;
        println!("Перевод {} USDT на адрес {}", amount, to);
        true
    } else {
        println!("❌ Недостаточно средств");
        false
    }
}
```

### Пример 3: проверка нескольких кошельков

```rust
struct Wallet {
    address: String,
    balance: f64,
}

fn main() {
    let wallets = vec![
        Wallet { address: String::from("0xAlice"), balance: 100.0 },
        Wallet { address: String::from("0xBob"), balance: 50.0 },
        Wallet { address: String::from("0xCharlie"), balance: 200.0 },
    ];
    
    // ✅ Много неизменяемых ссылок — можно!
    let first = &wallets[0];
    let second = &wallets[1];
    
    println!("Первый: {}, Второй: {}", first.balance, second.balance);
    
    // ❌ Нельзя взять &mut при &T
    // let mut_ref = &mut wallets[2]; // ОШИБКА!
}
```

## 6️⃣ Ошибки новичков и безопасные варианты 🛡️

### ❌ Ошибка №1: Забыл, что move — это не copy

```rust
// ❌ НЕБЕЗОПАСНО: потеря данных
fn main() {
    let v = vec![1, 2, 3];
    process(v);
    println!("{:?}", v); // ОШИБКА! v перемещён
}

fn process(v: Vec<i32>) {
    println!("{:?}", v);
}

// ✅ БЕЗОПАСНО: используем ссылку
fn main() {
    let v = vec![1, 2, 3];
    process(&v);
    println!("{:?}", v); // ✅ Работает!
}

fn process(v: &[i32]) {
    println!("{:?}", v);
}
```

### ❌ Ошибка №2: Две изменяемые ссылки

```rust
// ❌ НЕБЕЗОПАСНО: data race (гонка данных)
let mut s = String::from("Hello");
let r1 = &mut s;
let r2 = &mut s; // ОШИБКА! две &mut ссылки
println!("{}, {}", r1, r2);

// ✅ БЕЗОПАСНО: используем последовательно
let mut s = String::from("Hello");
{
    let r1 = &mut s;
    r1.push_str(" world");
} // r1 выходит из области видимости

let r2 = &mut s;
r2.push_str("!!!");
println!("{}", s); // "Hello world!!!"
```

### ❌ Ошибка №3: Возврат ссылки на локальную переменную

```rust
// ❌ НЕБЕЗОПАСНО: висячая ссылка
fn get_ref() -> &String {
    let s = String::from("Привет");
    &s // ОШИБКА! s умрёт после выхода из функции
}

// ✅ БЕЗОПАСНО: возвращаем владение
fn get_string() -> String {
    let s = String::from("Привет");
    s // владение переходит вызывающему
}
```

## 7️⃣ Время жизни (Lifetimes) — простое объяснение

### Что такое lifetime?

Lifetime (время жизни) — это **метка**, которая говорит компилятору:
«Ссылка `&T` будет жить не дольше, чем данные, на которые она указывает».

### Зачем это нужно?

Чтобы предотвратить **висячие ссылки** (dangling references) — когда ссылка указывает на уже удалённые данные.

### Как это выглядит в коде?

```rust
// Пример: функция возвращает ссылку на
// самый длинный из двух строковых срезов
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

fn main() {
    let s1 = String::from("длинная строка");
    let result;
    {
        let s2 = String::from("короткая");
        result = longest(&s1, &s2); // ✅ s2 живёт дольше, чем result
        println!("Самая длинная: {}", result);
    }
    // println!("{}", result); // ❌ если бы! s2 уже умерла
}
```

### 🧠 Запоминалка

> `'a` — это просто «метка». Она означает: «все ссылки с меткой `'a` живут одинаковое время».

### 💡 Совет опытного разработчика №4

> В 90% случаев Rust сам выводит lifetimes (это называется **элизия** — elision).
> Явно писать `'a` нужно только когда:
> 1. Функция принимает несколько ссылок и возвращает одну из них
> 2. В структуре есть поля-ссылки

## 8️⃣ Срезы (Slices)

### Что такое срез?

Срез (`&str`, `&[i32]`) — это **ссылка на часть данных** (не владеющая, просто «окошко»).

```rust
let text = String::from("Привет, мир!");
let hello = &text[0..6]; // "Привет" — срез первых 6 байт
let world = &text[7..12]; // "мир!"

println!("{}", hello);
println!("{}", world);
println!("{}", text);   // ✅ Оригинал всё ещё жив
```

### Зачем срезы?

```rust
// ❌ ПЛОХО: передаём владение String, потом возвращаем
fn first_word(s: String) -> (String, usize) {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return (s, i); // возвращаем строку обратно
        }
    }
    (s, s.len())
}

// ✅ ХОРОШО: используем срез
fn first_word(s: &str) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[..i];
        }
    }
    &s[..]
}
```

## 9️⃣ Сводка советов опытного разработчика 🧠

| № | Совет | Суть |
|---|-------|------|
| 1️⃣ | **Ссылки вместо move** | Если функция только читает — принимай `&T`, а не `T` |
| 2️⃣ | **Не злоупотребляй clone** | `.clone()` — костыль. Изучи `&` |
| 3️⃣ | **Сужай область ссылок** | Если ошибка «already borrowed» — вынеси в блок `{ }` |
| 4️⃣ | **Trust the compiler** | Rust не даст скомпилировать код с гонками или висячими ссылками |
| 5️⃣ | **&str вместо &String** | `&str` — универсальнее, принимает и `&String`, и `&str` |
| 6️⃣ | **Не бойся lifetimes** | В 90% случаев компилятор выводит их сам |
| 7️⃣ | **Сначала &T, потом &mut T** | Сначала прочитай, потом меняй — последовательно |
| 8️⃣ | **Срезы — это ссылки** | Они подчиняются тем же правилам заимствования |

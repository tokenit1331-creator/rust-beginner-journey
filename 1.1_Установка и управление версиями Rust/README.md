# Управление версиями Rust: `rustup`, `rustc`, `cargo`, переключение stable/nightly

## 1. Введение: почему нельзя просто установить `rustc` один раз?

Rust активно развивается: каждые 6 недель выходит новая стабильная версия, а ещё есть **бета-канал** (тестирование будущего стабильного релиза) и **ночной канал (nightly)**, где появляются экспериментальные возможности. Для большинства проектов достаточно `stable`. Однако, если вы пишете операционную систему, драйвер, встраиваемое ПО или используете нестабильные функции (например, встроенный ассемблер `asm!`, кастомные аллокаторы, `no_std` со специальными атрибутами), вам может понадобиться `nightly`.

`rustup` — официальный инструмент, который управляет установкой и переключением между этими каналами, а также версиями самого `rustc`, `cargo`, стандартной библиотеки и дополнительными компонентами.

---

## 2. Основные компоненты

| Компонент | Назначение |
|-----------|------------|
| **`rustc`** | Компилятор Rust. Обычно не вызывается напрямую, а через `cargo`. |
| **`cargo`** | Система сборки, менеджер пакетов. |
| **`rustup`** | Установщик и менеджер версий. Он позволяет держать несколько каналов (stable, beta, nightly) и переключаться между ними глобально или по проектам. |
| **`rust-std`** | Стандартная библиотека для целевой платформы. Например, `x86_64-unknown-linux-gnu` или `thumbv7m-none-eabi`. |
| **`rust-src`** | Исходники стандартной библиотеки (нужны для `rust-analyzer` и отладки). |
| **`rust-docs`** | Локальная документация (`rustup doc`). |

---

## 3. Установка `rustup`

На **Linux/macOS** (в терминале):

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

На **Windows** скачайте `rustup-init.exe` с официального сайта.

После установки перезагрузите терминал или выполните `source ~/.cargo/env`. Проверьте:

```bash
rustup --version
cargo --version
rustc --version
```

По умолчанию устанавливается **stable** канал.

---

## 4. Каналы (channels) Rust

Существует три канала обновлений:

| Канал | Стабильность | Когда использовать |
|-------|--------------|---------------------|
| **`stable`** | Полностью стабильный, обратная совместимость гарантирована | 99% проектов |
| **`beta`** | Следующий стабильный релиз, проходит тестирование | Проверка совместимости библиотек |
| **`nightly`** | Ежедневные сборки, могут содержать экспериментальные и сломанные функции | Разработка ядер, драйверов, `no_std` с нестабильным API, тестирование новых фич |

## 5. Управление каналами и версиями

### 5.1 Установка другого канала

```bash
rustup install nightly           # установить ночную сборку
rustup install beta
rustup install stable
```

### 5.2 Просмотр установленных версий

```bash
rustup toolchain list
```

Пример вывода:
```
stable-x86_64-unknown-linux-gnu (default)
beta-x86_64-unknown-linux-gnu
nightly-x86_64-unknown-linux-gnu
```

### 5.3 Переключение глобального канала по умолчанию

```bash
rustup default stable
rustup default nightly
```

### 5.4 Переключение канала для конкретного проекта (переопределение в каталоге)

Внутри проекта создайте файл `rust-toolchain.toml` (или `rust-toolchain` с именем канала). Например:

```toml
[toolchain]
channel = "nightly"
targets = ["thumbv7m-none-eabi"]
components = ["rust-src", "rustfmt", "clippy"]
```

Теперь при нахождении в этом каталоге `cargo`, `rustc` автоматически используют `nightly`. Это идеально для проектов, требующих нестабильных фич, не влияя на глобальные настройки.

Также можно установить локально:

```bash
rustup override set nightly   # в текущем каталоге
```

Проверить переопределение: `rustup override list`.

### 5.5 Обновление всех каналов

```bash
rustup update
```

Обновляет `rustup` и все установленные тулчейны.

## 6. Важные компоненты для системного программирования

При разработке ядра ОС или драйвера часто используются:

- **`rust-src`** — позволяет `rust-analyzer` показывать документацию по внутренностям `core`, `alloc`.
- **`llvm-tools-preview`** — инструменты вроде `llvm-objdump`, `llvm-nm`, полезные для анализа бинарных файлов без стандартной библиотеки.
- **`rust-std` для целевой платформы** — например, `thumbv7m-none-eabi` для ARM Cortex-M.

Установка:

```bash
rustup component add rust-src
rustup component add llvm-tools-preview
rustup target add thumbv7m-none-eabi
```

## 7. Зачем нужен `nightly` в реальных проектах?

### 7.1 Ядро ОС (например, Redox)

Ядро работает в окружении без стандартной библиотеки (`no_std`). Оно использует нестабильные фичи:

- `#![no_std]` — отключает стандартную библиотеку.
- `#![feature(asm)]` (устарело, теперь `asm!`).
- `#![feature(alloc_error_handler)]` — кастомный обработчик ошибок выделения памяти.
- `#![feature(lang_items)]` — необходимые символы для паники, OOM.
- `#![feature(const_fn_floating_point)]` — константные вычисления с плавающей точкой.

Пример `main.rs` для ядра:

```rust
#![no_std]
#![no_main]
#![feature(asm_const)]

use core::arch::asm;

#[no_mangle]
pub extern "C" fn _start() -> ! {
    unsafe {
        asm!("hlt");
    }
    loop {}
}
```

Такой код **требует nightly**.

### 7.2 Драйверы и встраиваемые системы

Для микроконтроллеров часто нужно:

- `#![feature(used_with_arg)]` для секций памяти.
- `#![feature(pointer_is_aligned)]` для низкоуровневых проверок.

### 7.3 Кастомные аллокаторы

В `stable` нельзя определить свой глобальный аллокатор (пока). В `nightly` есть `#![global_allocator]` с нестабильным трейтом `Allocator`.

### 7.4 Тестирование новых возможностей языка

Библиотеки иногда предоставляют функциональность, доступную только на `nightly`, с условной компиляцией.

## 8. Установка целевых платформ (targets)

При кросс-компиляции (например, для ARM) нужно установить соответствующий `target`. Без него `cargo build --target=...` не найдёт стандартную библиотеку.

```bash
rustup target add thumbv7m-none-eabi
rustup target add x86_64-unknown-none   # для ядра x86_64
```

Список всех доступных целей: `rustc --print target-list`.

## 9. Пример: создание проекта для ядра на nightly

1. Установить `nightly`:

```bash
rustup install nightly
```

2. Создать проект:

```bash
cargo new my_kernel --bin
cd my_kernel
```

3. Переопределить канал в текущей папке:

```bash
rustup override set nightly
```

4. Добавить файл `rust-toolchain.toml` для надёжности (чтобы другие разработчики использовали ту же версию):

```toml
[toolchain]
channel = "nightly-2025-04-01"   # фиксированная дата для воспроизводимости
components = ["rust-src"]
targets = ["x86_64-unknown-none"]
```

5. В `Cargo.toml` добавить:

```toml
[profile.dev]
panic = "abort"

[profile.release]
panic = "abort"
```

6. Написать код `src/main.rs` с `#![no_std]` и `#![no_main]`.

## 10. Устаревшее и современное

- Раньше использовали `cargo build -Z` флаги для нестабильных возможностей. Теперь для этого либо `nightly`, либо переменные окружения `RUSTC_BOOTSTRAP=1` (только для компилятора, не рекомендуется для повседневной работы).
- `rustup` также позволяет установить конкретную дату nightly: `rustup install nightly-2025-04-01`. Это нужно для воспроизводимых сборок.

---

## 11. Проблемы и их решение

- **Ошибка: `can't find crate for 'core'`** → скорее всего, вы пытаетесь собрать `no_std` проект на `stable`. Решение: переключитесь на `nightly`.
- **Ошибка: `target not installed`** → выполните `rustup target add <target>`.
- **Разные версии между проектами** → используйте `rustup override` или `rust-toolchain.toml`.
- **`rustfmt` или `clippy` недоступны в nightly** → установите компоненты отдельно: `rustup component add rustfmt --toolchain nightly`.

---

## 12. Итог

- `rustup` — центральный инструмент для управления версиями и каналами Rust.
- `stable` — для повседневной разработки.
- `nightly` — для системного программирования, ядер, драйверов, прототипов новых фич.
- `rustup override` и `rust-toolchain.toml` позволяют зафиксировать нужный канал для проекта без глобального переключения.
- Не забывайте устанавливать дополнительные компоненты (`rust-src`, `llvm-tools`, `targets`).

Теперь вы готовы настраивать окружение для любых проектов — от консольных утилит до операционных систем. 🚀

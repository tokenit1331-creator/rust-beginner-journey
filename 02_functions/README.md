## 1. Что такое функция?

**Функция** — это именованный блок кода, который принимает входные данные, выполняет действия и возвращает результат.

### 🎯 Для новичка: метафора

Представь **торговый автомат**:
- 🪙 Ты кладёшь монеты (**параметры/вход**)
- 🤖 Автомат обрабатывает (тело функции)
- 🥤 Выдаёт колу (**возвращаемое значение**)

```rust
fn buy_cola(money: f64) -> String {
    // money — вход (параметр)
    // String — выход (возвращаемое значение)
    String::from("🥤 Колу, пожалуйста!")
}
```

### 🤔 Зачем нужны функции?

| Без функции | С функцией |
|------------|------------|
| Повторяешь один код 10 раз | Написал 1 раз, вызвал 10 раз |
| Ошибка в одном месте → правишь везде | Ошибка → правишь в одном месте |
| 1000 строк в main() | main() — 10 строк, логика в функциях |

## 2. Анатомия функции

```rust
//   ↓       ↓           ↓            ↓
// ключ.слово  имя    параметры   возвращаемый тип
   fn     greet(name: String) -> String {
//          └─────────────────┘     ──────┬─────
//                 вход                выход
       format!("Привет, {}!", name)
   //   └──────────────────────────┘
   //          тело функции
   //          последнее выражение без ; = return
   }
```

### Разбор каждой части

| Часть | Что это | Обязательно? | Пример |
|-------|---------|:---:|--------|
| `fn` | Ключевое слово — говорит Rust "эй, тут функция" | ✅ | `fn` |
| Имя | Как будем вызывать | ✅ | `greet` |
| `()` | Скобки для параметров | ✅ | `(name: String)` |
| `-> Тип` | Стрелка + тип того, что возвращаем | ❌ | `-> String` |
| `{}` | Тело — код внутри | ✅ | `{ ... }` |

### Простейшая функция

```rust
fn say_hello() {
    // Нет параметров, нет возврата
    println!("Привет!");
}
```

## 3. Параметры и возвращаемые значения

### Без параметров, без возврата

```rust
fn show_menu() {
    println!("1. BTC  2. ETH  3. SOL");
    // ничего не возвращает (тип возврата = ())
}
```

### С параметрами, без возврата

```rust
fn show_balance(address: &str) {
    // принимает ссылку на строку (не забирает владение)
    println!("Баланс по адресу: {}", address);
}
```

### С параметрами и возвратом

```rust
fn add(a: i32, b: i32) -> i32 {
    a + b  // последнее выражение БЕЗ ; = return
}

// Можно и с return, но это длиннее:
fn add_with_return(a: i32, b: i32) -> i32 {
    return a + b; // ; можно, но не нужно
}
```

### Несколько возвратов через enum

```rust
enum TxStatus { Success, Failed, Pending }

fn check_tx(tx_id: &str) -> TxStatus {
    if tx_id.len() < 10 { return TxStatus::Failed; }
    TxStatus::Success // без ; = return
}
```

### Возврат ошибки через Result

```rust
fn parse_amount(raw: &str) -> Result<f64, String> {
    match raw.parse::<f64>() {
        Ok(val) if val > 0.0 => Ok(val),
        Ok(_) => Err("Сумма должна быть > 0".into()),
        Err(_) => Err("Не число".into()),
    }
}
```

## 4. Область видимости

Каждая функция создаёт **новую область видимости** (scope).

```rust
fn outer() {
    let x = 10;  // x живёт ТОЛЬКО внутри outer
    
    fn inner() {
        // x ЗДЕСЬ НЕ ВИДЕН!
        let y = 20; // y живёт ТОЛЬКО внутри inner
    }
    
    // y ЗДЕСЬ НЕ ВИДЕН!
}
```

### 🎯 Почему это важно?

Переменные внутри функции **умирают** после её завершения. Rust чистит память автоматически.

```rust
fn process() {
    let big_data = vec![0u8; 1_000_000]; // 1 МБ
    // ... работаем с big_data ...
} // ← big_data УДАЛЯЕТСЯ здесь!
// Память освобождена — утечки нет!
```

## 5. Выражения vs Операторы

Это **САМАЯ ВАЖНАЯ** тема для понимания return в Rust.

### Определения

| Конструкция | Что делает | Пример |
|------------|-----------|--------|
| **Оператор** | Выполняет действие, НЕ возвращает значение | `let x = 5;` |
| **Выражение** | Возвращает значение | `5 + 3` (возвращает 8) |

### Ключевой момент

**Последнее выражение без `;` — это возврат.**

```rust
fn correct() -> i32 {
    let x = 5;     // оператор (есть ;)
    let y = 10;    // оператор (есть ;)
    x + y          // выражение БЕЗ ; → return x + y
}

// АНАЛОГИЧНО через return:
fn also_correct() -> i32 {
    let x = 5;
    let y = 10;
    return x + y;
}
```

### Типичная ошибка новичка

```rust
fn broken() -> i32 {
    let x = 5;
    x + 10;  // ❌ ; превращает выражение в оператор!
              // Ошибка: expected i32, found ()
}

fn fixed() -> i32 {
    let x = 5;
    x + 10   // ✅ без ; — возвращает 15
}
```

## 6. Передача по ссылке vs по значению

### По значению (move) — владение переходит

```rust
fn process(data: Vec<i32>) -> i32 {
    // data — теперь ВЛАДЕНИЕ функции
    data.iter().sum()
} // data УНИЧТОЖАЕТСЯ здесь

// После вызова data НЕДОСТУПНА:
let my_data = vec![1, 2, 3];
let sum = process(my_data);
// println!("{:?}", my_data); // ❌ ОШИБКА! data перемещена
```

### По ссылке (&) — только чтение

```rust
fn process(data: &[i32]) -> i32 {
    // data — ССЫЛКА (заимствование)
    data.iter().sum()
} // ссылка УМИРАЕТ, но data остаётся

let my_data = vec![1, 2, 3];
let sum = process(&my_data);
println!("{:?}", my_data); // ✅ ВСЁ ЕЩЁ ДОСТУПНА!
```

### По мутабельной ссылке (&mut) — можно менять

```rust
fn add_token(tokens: &mut Vec<String>, token: String) {
    tokens.push(token);
}

let mut portfolio = vec!["BTC".to_string()];
add_token(&mut portfolio, "ETH".to_string());
println!("{:?}", portfolio); // ["BTC", "ETH"]
```

## 7. Замыкания (Closures)

Замыкание — это анонимная функция, которая "захватывает" переменные из внешней среды.

```rust
let fee = 0.01; // комиссия
let calc_fee = |amount: f64| -> f64 {
    amount * fee  // fee — ЗАХВАЧЕНА извне!
};

println!("{}", calc_fee(100.0)); // 1.0
```

### Три типа замыканий

| Трейт | Как захватывает | Сколько раз можно вызвать |
|-------|----------------|:-------------------------:|
| `Fn` | По ссылке (`&`) | ♾️ Любой раз |
| `FnMut` | По мутабельной ссылке (`&mut`) | ♾️ Любой раз (через &mut) |
| `FnOnce` | По значению (move) | 1️⃣ Только один раз |

```rust
let rate = 1.2;
let fn_closure = |x| x * rate;          // Fn — только читает

let mut count = 0;
let mut fnmut_closure = || { count += 1; }; // FnMut — меняет

let data = vec![1, 2, 3];
let fnonce_closure = move || { println!("{:?}", data); }; // FnOnce — забирает
```

## 8. Функции как аргументы

Можно передавать функции в другие функции как параметры.

```rust
// f — параметр-функция: принимает i32, возвращает i32
fn apply_twice(f: impl Fn(i32) -> i32, x: i32) -> i32 {
    f(f(x)) // применяем f дважды
}

let double = |x| x * 2;
println!("{}", apply_twice(double, 5)); // 20
```

### fn pointer vs замыкание

```rust
fn add_one(x: i32) -> i32 { x + 1 }

// fn pointer — обычная функция
fn use_fn_ptr(f: fn(i32) -> i32, x: i32) -> i32 { f(x) }

// Замыкание — может захватывать переменные
fn use_closure(f: impl Fn(i32) -> i32, x: i32) -> i32 { f(x) }
```

## 9. Обобщённые функции (Generics)

Одна функция для всех типов.

```rust
fn first<T>(list: &[T]) -> Option<&T> {
    // T — любой тип
    list.first()
}

// Работает с любым типом:
first(&[1, 2, 3]);        // i32
first(&["a", "b", "c"]);  // &str
first(&[1.0, 2.0]);       // f64
```

### С ограничением (trait bound)

```rust
fn find_max<T: PartialOrd>(items: &[T]) -> Option<&T> {
    // T должен поддерживать сравнение (PartialOrd)
    let mut max = items.first()?;
    for item in items {
        if item > max { max = item; }
    }
    Some(max)
}
```

## 10. Идиомы и лучшие практики

### ✅ Ранний return для ошибок

```rust
fn process(order: &Order) -> Result<(), String> {
    if order.price <= 0.0 { return Err("price <= 0".into()); }
    if order.amount <= 0.0 { return Err("amount <= 0".into()); }
    // логика...
    Ok(())
}
```

### ✅ Комбинаторы вместо циклов

```rust
let total: f64 = transactions.iter()
    .filter(|t| t.status == "success")
    .map(|t| t.amount)
    .sum();
```

### ✅ Одна функция — одна ответственность

```rust
// ПЛОХО: валидирует И логирует
fn process(tx: &str) -> bool { ... }

// ХОРОШО: разделено
fn validate(tx: &str) -> bool { ... }
fn log(tx: &str) { ... }
```

### ✅ Маленькие функции — это нормально

Функция на 3 строки — отлично. На 100 строк — переделывай.

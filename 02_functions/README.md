## 1. Что такое функция?

Функция — это именованный блок кода, который принимает входные данные, выполняет действия и возвращает результат.

### Метафора
Функция — это станок на заводе. Вы кладёте сырьё (параметры), станок обрабатывает (тело функции), вы получаете готовую деталь (возвращаемое значение).

```rust
fn add(x: i32, y: i32) -> i32 {
    x + y
}
```

## 2. Анатомия функции

```rust
//   ^    ^       ^         ^
//   |    |       |         |
// ключ имя  параметры  возвращаемый тип
fn greet(name: String) -> String {
    format!("Привет, {}!", name)
}
```

| Часть | Описание | Пример |
|-------|----------|--------|
| `fn` | Ключевое слово — объявляет функцию | `fn` |
| Имя | Как будем вызывать | `greet` |
| Параметры | Входные данные | `(name: String)` |
| `-> Тип` | Что возвращаем | `-> String` |
| Тело | Что делает | `{ ... }` |

## 3. Параметры и возвращаемые значения

### Без параметров, без возврата
```rust
fn say_hello() {
    println!("Привет!");
}
```

### С параметрами и возвратом
```rust
fn add(a: i32, b: i32) -> i32 {
    a + b  // без ; = return
}
```

### Ранний return
```rust
fn validate_age(age: i32) -> Result<(), String> {
    if age < 0 {
        return Err("Возраст не может быть отрицательным".into());
    }
    Ok(())
}
```

## 4. Область видимости

Каждая функция создаёт новую область видимости. Переменные внутри функции не видны снаружи.

```rust
fn outer() {
    let x = 10;
    fn inner() {
        // x здесь НЕ ВИДЕН!
        let y = 20;
    }
    // y здесь НЕ ВИДЕН!
}
```

## 5. Выражения vs Операторы

Это самое важное для понимания return в Rust.

| Конструкция | Тип | Пример |
|------------|-----|--------|
| **Оператор** | Выполняет действие, не возвращает значение | `let x = 5;` |
| **Выражение** | Возвращает значение | `x + 5` |

**Ключевой момент:** последнее выражение без `;` — это возврат.

```rust
fn correct() -> i32 {
    let x = 5;
    x + 10  // ✅ возвращает 15
}

fn wrong() -> i32 {
    let x = 5;
    x + 10; // ❌ ошибка! выражение стало оператором
}
```

## 6. Передача по ссылке vs по значению

### По значению (move)
Владение переходит в функцию.

```rust
fn process(data: Vec<i32>) -> i32 {
    data.iter().sum()
}
```

### По ссылке (&)
Функция одалживает данные.

```rust
fn process(data: &[i32]) -> i32 {
    data.iter().sum()
}
```

### По мутабельной ссылке (&mut)
Функция может изменить данные.

```rust
fn add_one(data: &mut Vec<i32>) {
    data.push(1);
}
```

## 7. Замыкания (Closures)

Замыкание — это анонимная функция, которая может захватывать переменные из внешней среды.

```rust
let multiplier = 3;
let multiply = |x: i32| x * multiplier;
println!("{}", multiply(5)); // 15
```

| Трейт | Что делает | Когда используется |
|-------|-----------|-------------------|
| `Fn` | Захватывает по ссылке (&) | Можно вызывать много раз |
| `FnMut` | Захватывает по &mut | Можно изменять захваченное |
| `FnOnce` | Захватывает по значению | Можно вызвать только 1 раз |

## 8. Функции как аргументы

Можно передавать функции в другие функции.

```rust
fn apply_twice(f: impl Fn(i32) -> i32, x: i32) -> i32 {
    f(f(x))
}

let double = |x| x * 2;
apply_twice(double, 5); // 20
```

## 9. Обобщённые функции (Generics)

Позволяют писать одну функцию для разных типов.

```rust
fn first<T>(list: &[T]) -> Option<&T> {
    list.first()
}
```

## 10. Идиомы и лучшие практики

### ✅ Ранний return для ошибок
```rust
fn parse_tx(raw: &str) -> Result<Tx, String> {
    let parts: Vec<&str> = raw.split(',').collect();
    if parts.len() != 3 {
        return Err("Нужно 3 поля".into());
    }
    Ok(tx)
}
```

### ✅ Идиомы
- Одна функция — одна ответственность
- impl Trait для простоты
- Комбинаторы вместо match
- Маленькие функции — это нормально
